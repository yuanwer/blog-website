# 简单易懂的 Promise 用法解析

Promise 是 ES6 中的重要特性，它可以让我们更优雅地处理异步请求，但它有不少的方法和特点，容易让人困惑，这篇文章就是为了清楚的说明 Promise 的用法。

TIP

在官方的 Promise 出现之前社区中已存在多种异步解决方案。有些方案在使用方式上与后来的 Promise 有一些相似之处，比如 jQuery 的 Deferred 对象。然而，本文将专注于介绍 ES6 中的 Promise，不会深入探讨其他方案。

# Promise 的简介

# 早期的异步编程方案

在 Promise 出现之前，前端开发流行的异步编程方案主要是回调函数。

假设我们有一个网络请求的方法，叫做 getUser ，用于获取用户数据，代码如下：

function getUser(success, error) {
  // 模拟网络请求
  setTimeout(() => {
    const user = {
      name: '张三',
      age: 18,
    }

    // 随机，偶数成功，奇数失败
    const random = Math.floor(Math.random() * 10)
    if (random % 2 === 0) {
      success(user)
    } else {
      error('请求失败')
    }
  }, 1000)
}

当我们使用它时，需要传入两个回调函数，一个是成功的回调函数，一个是失败的回调函数，代码如下：

getUser(
  user => {
    console.log(user)
  },
  error => {
    console.log(error)
  },
)

乍一看，这样的代码还是比较简单的，但是当我们需要多次请求数据，并且后一个请求的参数依赖于前一个请求的结果时，代码就会变得非常难看，比如下面这个例子：

getUser(
  user => {
    // 获取用户的订单
    getOrder(
      order => {
        // 获取订单的商品
        getGoods(
          goods => {
            console.log(goods)
          },
          error => {
            console.log(error)
          },
          order.id,
        )
      },
      error => {
        console.log(error)
      },
      user.id,
    )
  },
  error => {
    console.log(error)
  },
)

是不是有点毛骨悚然了，这就是所谓的回调地狱，代码的缩进越来越多，越来越深，越来越难看，这就是回调函数的缺点之一。

回调函数的方案还有其他问题：

  1. 程序员需要自己去设计回调函数的一切，包括名称、如何使用等等，这就需要程序员有一定的经验，否则很容易出错。
  2. 不同的程序员，或不同的框架，设计出来的方案是不同的，我们在使用别人的代码前，需要耐心查看文档甚至源代码，才能知道如何使用。

# 什么是 Promise

Promise 译为“承诺”，它是异步编程的一种解决方案，比传统的回调函数和事件更合理和更强大。

先来一段简答的代码：

const promise = new Promise((resolve, reject) => {
  // 调用 resolve 函数，会将 Promise 的状态改为 fulfilled，触发 then 方法传入的回调函数
  resolve('成功')
  // 调用 reject 函数，会将 Promise 的状态改为 rejected，触发 catch 方法传入的回调函数
  reject('失败')
})

promise.then(
  value => {
    console.log(value)
  },
  error => {
    console.log(error)
  },
)

让我们解读上面的代码：

  • Promise 是一个类，使用它需要使用new关键字
  • 我们传入了一个回调函数作为参数，可以称它为 executor
  • executor 函数会被立即执行，并自动传入两个参数，分别是 resolve 和 reject，这两个参数也是函数
    • resolve 函数用于将 Promise 的状态改为成功，并将结果作为参数传递出去，这会触发 then 方法传入的回调函数
    • reject 函数用于将 Promise 的状态改为失败，并将结果作为参数传递出去，这会触发 catch 方法传入的回调函数

# Promise 的状态

从上面的代码，可以看出 Promise 有三种状态，分别是：

  • pending：待定，Promise 的初始状态，既没成功，也没失败
  • fulfilled：成功，当调用了 resolve 函数时，状态会变为 fulfilled，此时会调用 then 方法传入的回调函数
  • rejected：失败，当调用了 reject 函数时，状态会变为 rejected，此时会调用 catch 方法传入的回调函数

# 使用 Promise 来重构 getUser 方法

我们可以使用 Promise 来重构上面的 getUser 方法，代码如下：

function getUser() {
  return new Promise((resolve, reject) => {
    // 模拟网络请求
    setTimeout(() => {
      const user = {
        name: '张三',
        age: 18,
      }

      // 随机，偶数成功，奇数失败
      const random = Math.floor(Math.random() * 10)
      if (random % 2 === 0) {
        resolve(user)
      } else {
        reject('请求失败')
      }
    }, 1000)
  })
}

const promise = getUser()
promise.then(
  user => {
    console.log(user)
  },
  error => {
    console.log(error)
  },
)

你可能会说，这不还是回调函数吗，这和上面的回调函数有什么区别呢？

确实，这和上面的回调函数没有本质的区别，但是 Promise 的优势在于，它可以将回调函数的嵌套，改为链式调用，还记得上面毛骨悚然的回调地狱吗？现在我们可以使用 Promise 来重构它，代码如下：

// 获取用户
getUser()
  .then(user => {
    // 获取用户的订单
    return getOrder(user.id)
  })
  .then(order => {
    // 获取订单的商品
    return getGoods(order.id)
  })
  .then(goods => {
    // 打印商品
    console.log(goods)
  })
  .catch(error => {
    // 打印错误
    console.log(error)
  })

# Promise 的对象方法

# Executor 函数

Executor 函数是在创建 Promise 时需要传入的一个回调函数，它会被立即执行，并且自动传入两个参数：

const promise = new Promise((resolve, reject) => {
  // resolve 和 reject 都是函数
})

我们会在 Executor 函数中确定 Promise 的状态，如果我们认为 Promise 成功了，就调用 resolve 函数，如果我们认为 Promise 失败了，就调用 reject 函数。

需要注意的是，一旦 Promise 的状态被确定下来，就不能再改变了，这也是 Promise 的特点之一。

当我们调用了 resolve 函数时，传入的参数如果是非 Promise 对象，那么会将 Promise 的状态改为 fulfilled，之后就算我们再次调用了 reject 函数，也不会改变 Promise 的状态，代码如下：

const promise = new Promise((resolve, reject) => {
  resolve('成功')
  reject('失败')
}).then(
  value => {
    // 会打印“成功”
    console.log(value)
  },
  error => {
    // 不会执行
    console.log(error)
  },
)

并不是 reject 函数没有执行，而是 resolve 函数已经将 Promise 的状态改为 fulfilled 了，之后再调用 reject 函数，也不会改变 Promise 的状态。

另一种情况是，在调用 resolve 函数时，传入的参数是另一个 Promise 对象，那么这个新的 Promise 对象的状态会决定当前 Promise 对象的状态，代码如下：

const promise = new Promise((resolve, reject) => {
  resolve(new Promise((resolve, reject) => reject('失败')))
}).then(
  value => {
    // 不会执行
    console.log(value)
  },
  error => {
    // 会打印“失败”
    console.log(error)
  },
)

如果调用了 resolve 函数，传入的是一个普通的 js 对象，并且对象当中有实现了 then 方法，那么这个对象也会被当做 Promise 对象来处理，代码如下：

const promise = new Promise((resolve, reject) => {
  resolve({
    then(_resolve, _reject) {
      _resolve('成功')
    },
  })
}).then(
  value => {
    // 会打印“成功”
    console.log('value: ', value)
  },
  error => {
    // 不会执行
    console.log('error: ', error)
  },
)

# then 方法

then 方法是 Promise 的实例方法，实际上它是 Promise 原型对象上的方法，即 Promise.prototype.then。

# then 方法的两个参数

虽然上面我们在使用 then 方法时，只传入了一个参数，但实际上 then 方法有两个参数，第一个参数是成功的回调函数，第二个参数是失败的回调函数，代码如下：

const promise = new Promise((resolve, reject) => {
  reject('失败')
}).then(
  value => {
    console.log('value: ', value)
  },
  error => {
    console.log('error: ', error)
  },
)

这个代码等同于：

const promise = new Promise((resolve, reject) => {
  reject('失败')
})
  .then(value => {
    console.log('value: ', value)
  })
  .catch(error => {
    console.log('error: ', error)
  })

我还是更加推荐使用 catch 方法，因为这样可以将错误的处理和成功的处理分开，代码更加清晰。

# then 方法的多次调用

一个 Promise 对象的 then 方法可以多次调用，当状态变为 fulfilled 时，所有的回调函数都会被调用，代码如下：

const promise = new Promise((resolve, reject) => {
  resolve('成功')
})

promise.then(value => {
  console.log('value1: ', value)
})

promise.then(value => {
  console.log('value2: ', value)
})

promise.then(value => {
  console.log('value3: ', value)
})

最后打印的结果：

value1: 成功
value2: 成功
value3: 成功

# then 方法的返回值

then 方法其实是有返回值的，返回的是一个新的 Promise 对象，所以我们才能进行链式调用。

既然返回的是一个新的 Promise 对象，那这个 Promise 对象的状态是什么呢？

这需要分情况来讨论：

当 then 方法中的回调函数正在执行时，返回的 Promise 对象的状态是 pending，代码如下：

const promise = new Promise((resolve, reject) => {
  resolve('成功')
})

const promise2 = promise.then(value => {})

console.log(promise2) // Promise { <pending> }

当 then 方法中的回调函数返回了一个结果时，返回的 Promise 对象的状态是 fulfilled，代码如下：

const promise = new Promise((resolve, reject) => {
  resolve('成功')
})

const promise2 = promise.then(value => {
  return '成功2'
})

promise2.then(value => {
  console.log(value) // 成功2
})

并且回调函数返回的结果会作为 promise2 的 resolve 函数的参数。

那如果 then 的回调函数返回的不是“成功 2”，而是一个 Promise 对象呢？

那么这个 Promise 对象的状态会决定 promise2 的状态。

当 then 方法抛出了一个异常时，返回的 Promise 对象的状态是 rejected，代码如下：

const promise = new Promise((resolve, reject) => {
  resolve('成功')
})

const promise2 = promise.then(value => {
  throw new Error('失败')
})

promise2.then(
  value => {
    console.log(value)
  },
  error => {
    console.log(error) // Error: 失败
  },
)

# catch 方法

catch 方法是 Promise 的实例方法，它也是放在 Promise 原型对象上的方法，即 Promise.prototype.catch。

它也可以多次调用，每次调用时传入 reject 的回调函数，当 Promise 的状态变为 rejected 时，所有的回调函数都会被调用，代码如下：

const promise = new Promise((resolve, reject) => {
  reject('失败')
})

promise.catch(error => {
  console.log('error1: ', error)
})

promise.catch(error => {
  console.log('error2: ', error)
})

promise.catch(error => {
  console.log('error3: ', error)
})

最后打印的结果：

error1: 失败
error2: 失败
error3: 失败

# catch 方法的返回值

catch 方法也是有返回值的，和 then 方法一样，返回的是一个新的 Promise 对象，所以我们才能继续链式调用 then 或 catch 方法。

先上一段代码：

const promise = new Promise((resolve, reject) => {
  reject('失败')
})

promise
  .catch(error => {
    console.log('error1: ', error)
  })
  .catch(error => {
    console.log('error2: ', error)
  })
  .then(value => {
    console.log('value: ', value)
  })

最后打印的结果：

error1: 失败
value: undefined

为什么没有打印“error2: 失败”呢？

这是因为 catch 方法传入的回调函数执行后，默认状态依然是 fulfilled，所以后面的 then 方法会被调用，而不是 catch 方法。

如果我们希望 catch1 的回调函数执行后，继续执行 catch2 中的回调函数，我们需要抛出一个异常：

promise
  .catch(err => {
    throw new Error('error message')
  })
  .catch(err => {
    console.log('err2: ', err)
  })
  .then(value => {
    console.log('value: ', value)
  })

当然我们也可以通过返回一个 rejected 的 Promise 对象来实现：

promise
  .catch(err => {
    return Promise.reject('error message')
  })
  .catch(err => {
    console.log('err2: ', err)
  })
  .then(value => {
    console.log('value: ', value)
  })

两种的结果是一样的。

# finally 方法

finally 是 ES9（ES2018）中新增的方法，它也是 Promise 的实例方法，它也是放在 Promise 原型对象上的方法，即 Promise.prototype.finally。

finally 方法的用于指定不管 Promise 的状态是 fulfilled 还是 rejected，都会执行的回调函数，代码如下：

const promise = new Promise((resolve, reject) => {
  resolve('成功')
})

promise
  .then(value => {
    console.log('value: ', value)
  })
  .catch(error => {
    console.log('error: ', error)
  })
  .finally(() => {
    console.log('finally')
  })

finally 方法的回调函数不接收任何参数，因为它不关心 Promise 的状态。

finally 一般用于指定一些收尾工作，比如在 execute 函数中开启了一个 loading，那么不管 Promise 的状态是 fulfilled 还是 rejected，我们都需要关闭 loading，这时就可以使用 finally 方法。

# Promise 的类方法

上面说的 then、catch、finally 都是 Promise 的实例方法，它们是放在 Promise 原型对象上的方法，下面要说的是 Promise 的类方法，它们是放在 Promise 构造函数上的方法。

实例方法和类方法的区别可以简单理解为：实例方法是通过实例来调用的，而类方法是通过类来调用的。这属于类的知识，在这里不做过多的解释。

# Promise.resolve 方法

假设我手中已经有一个结果了，但我希望将它包装为一个 Promise 对象来使用，这时就可以使用 Promise.resolve 方法，代码如下：

const promise = Promise.resolve('成功')

// 等同于
const promise = new Promise((resolve, reject) => {
  resolve('成功')
})

# Promise.resolve 参数是一个 Promise 对象

如果 Promise.resolve 方法的参数是一个 Promise 对象，那么它会直接返回这个 Promise 对象，代码如下：

const promise1 = new Promise((resolve, reject) => {
  resolve('成功')
})

const promise2 = Promise.resolve(promise1)

console.log(promise1 === promise2) // true

# Promise.resolve 参数是 js 对象，但对象中有 then 方法

如果 Promise.resolve 方法的参数是一个 js 对象，但对象中有 then 方法，那么它会将这个对象当做 Promise 对象来处理，代码如下：

const promise = Promise.resolve({
  then(resolve, reject) {
    resolve('成功')
  },
})

promise.then(value => {
  console.log(value) // 成功
})

# Promise.resolve 参数是非对象，或者对象中没有 then 方法

如果 Promise.resolve 方法的参数是非对象，或者对象中没有 then 方法，那么这个参数会作为成功的结果：

const promise = Promise.resolve('成功')

promise.then(value => {
  console.log(value) // 成功
})

# Promise.reject 方法

Promise.reject 方法的作用是返回一个状态为 rejected 的 Promise 对象，代码如下：

const promise = Promise.reject('失败')

// 等同于
const promise = new Promise((resolve, reject) => {
  reject('失败')
})

reject 方法的参数无论传什么，都会直接作为 Promise 的失败结果，传递给 catch 方法的回调函数。
