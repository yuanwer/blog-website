# 原型和原型链

原型和原型链是 JavaScript 中精妙的设计，它为 JavaScript 提供了一种继承机制，使得 JavaScript 可以像面向对象语言一样使用。

另外，原型和原型链也是面试中经常被问到的知识点。

# 什么是原型

JavaScript 当中每个对象都有一个特殊的内置属性 [[prototype]]，这个属性的值是一个对象，这个对象就是原型对象。

如果以前完全没有接触过原型，可能会有点懵，没关系，一步一步来，先定义一个普通对象：

const obj = {
  name: 'obj',
}

console.log(obj.valueOf()) // { name: 'obj' }

我并没有给 obj 定义过 valueOf 方法，为什么 obj 可以调用这个方法呢，这个方法哪来的。

其实 valueOf 方法在 obj 对象的原型对象上，那怎么才能看见原型对象呢，有两种方式，第一种是访问对象的 __proto__ 属性，可以在控制台中输入 obj.__proto__ 查看 obj 的原型对象。

第二种方式是使用 Object.getPrototypeOf 方法，可以获取对象的原型对象：

Object.getPrototypeOf(obj)

两种方式获取的结果是一样的：

可以看到，obj 的原型对象上有很多方法，比如 valueOf、toString、hasOwnProperty 等等。

# 原型对象有什么用

作用主要有两个：

  1. 共享属性和方法
  2. 实现继承

# 共享属性和方法

当通过对象的 key 来获取对应的 value 时，会触发对象的[[Get]]操作，默认情况下，js 引擎会先在对象自身上查找这个属性，如果找到了就返回这个属性的值，如果没有找到，就会去对象的内置属性[[prototype]]指向的对象上查找这个属性。

说白了就是自己身上没有某个属性或方法也不要慌，原型对象上有也可以用。

知道了这个，就可以做一些有趣的事情，比如，我定义一个对象，为它添加很多实用的方法和属性，谁要使用我的方法和属性，就把我设置为原型对象就好了。

这里引出另一个问题：如何为一个对象设置原型对象？

同样有两种方式：

第一种是使用 __proto__ 属性，第二种是使用 Object.setPrototypeOf 方法。两种方式的结果是一致的。

const utils = {
  add: (a, b) => a + b,
  minus: (a, b) => a - b,
}

const obj1 = {
  name: 'obj1',
}

obj1.__proto__ = utils
console.log(obj1.add(1, 2)) // 3

const obj2 = {
  name: 'obj2',
}

Object.setPrototypeOf(obj2, utils)
console.log(obj2.minus(1, 2)) // -1

可以看见，obj1 和 obj2 都可以使用 utils 对象上的方法，而它们自身并不需要定义这些方法。即减少了程序员的工作量，也减少了内存的占用。

# 函数的原型

# 函数的 prototype

所有的函数身上都有一个 prototype 属性，这个属性的值是一个对象，这个对象就是函数的原型对象。

function foo() {}

console.log(foo.prototype) // {constructor: ƒ}

注意，函数本质上也是一个对象，ECMAScript 规范中把函数定义为一种可调用对象。但并不是因为它是一个对象，所以有 prototype 属性，而是因为它是一个函数，所以有 prototype 属性。这个很重要，很多人会弄混。

当你使用 new 关键字调用一个函数时，这个函数就是构造函数，有以下几个步骤：

  1. 在内存中创建一个新对象（空对象）
  2. 将新对象内部的 [[prototype]] 属性赋值为构造函数的 prototype 属性指向的对象
  3. 将构造函数的 this 指向这个新对象
  4. 执行构造函数内部的代码
  5. 如果构造函数手动返回了一个对象（任何对象，包括但不限于普通对象、时间对象···），那么这个对象就是 new 表达式的返回值，否则返回第一步创建的对象

这里主要看前两个步骤，第一步创建了一个新对象，第二步将新对象的 [[prototype]] 属性赋值为构造函数的 prototype 属性指向的对象。

也就是说：

function Person(name) {
  this.name = name
}

const lee = new Person('Lee')
const kim = new Person('Kim')

console.log(Person.prototype === lee.__proto__) // true
console.log(lee.__proto__ === kim.__proto__) // true

可以看见，lee 和 kim 的原型对象是同一个对象，这个对象就是 Person 函数的 prototype 属性指向的对象，因为它们的对象的内存地址是一样的。

上面的 Person 构造函数和 lee、kim 实例的关系如下图所示：

graph LR
    Person -->|prototype| Person.prototype
    Person -->|new| lee
    Person -->|new| kim
    lee -->|__proto__| Person.prototype
    kim -->|__proto__| Person.prototype

默认情况下，函数的 prototype 属性指向的对象只有一个属性，就是 constructor 属性，这个属性的值是这个函数本身。

function Person(name) {
  this.name = name
}

console.log(Person.prototype.constructor === Person) // true

我们可以往 prototype 对象上添加属性和方法，这样，所有通过这个构造函数创建的实例都可以使用这些属性和方法。

function Person(name) {
  this.name = name
}

Person.prototype.sayName = function () {
  console.log(`My name is ${this.name}`)
}

const lee = new Person('Lee')
lee.sayName() // My name is Lee

# 重写函数的原型对象

如果要在原型上添加很多的属性和方法，通常会重写原型对象。

function Person(name) {}

Person.prototype = {
  name: 'Person',
  sayName: function () {
    console.log(`My name is ${this.name}`)
  },
}

默认情况下，创建一个函数的同时，会创建它的 prototype 对象，这个对象中会有一个 constructor 属性，这个属性的值是这个函数本身。

但上面的代码，是给构造函数的 prototype 属性赋值了一个新的对象，这个新对象是单独创建的一个普通对象，它的 constructor 属性指向的是 Object 构造函数，而不是 Person 构造函数。

function Person(name) {}

Person.prototype = {
  name: 'Person',
  sayName: function () {
    console.log(`My name is ${this.name}`)
  },
}

console.log(Person.prototype.constructor === Person) // false
console.log(Person.prototype.constructor === Object) // true

因为这个新对象本质是通过 new Object() 创建的，所以它的原型对象是 Object.prototype，所以它的 constructor 属性指向的是 Object 构造函数。

const obj1 = {}
const obj2 = new Object()

console.log(obj1.__proto__ === Object.prototype) // true
console.log(obj2.__proto__ === Object.prototype) // true

console.log(obj1.constructor === Object) // true
console.log(obj2.constructor === Object) // true
console.log(obj1.constructor === obj2.constructor) // true

如果还是希望新对象的 constructor 属性指向 Person 构造函数，可以手动设置：

function Person(name) {}

Person.prototype = {
  constructor: Person,
  name: 'Person',
  sayName: function () {
    console.log(`My name is ${this.name}`)
  },
}

console.log(Person.prototype.constructor === Person) // true

需要注意的是，默认的函数原型对象上的 constructor 属性是不可枚举的，但手动设置的是可枚举的，如果不希望它是可枚举的，可以使用 Object.defineProperty 方法：

Object.defineProperty(Person.prototype, 'constructor', {
  value: Person,
  // 不可枚举
  enumerable: false,
})

# 原型链

在原型的部分已经知道，访问一个对象的属性，如果对象自身没有这个属性，就会去原型对象上查找，那如果原型上也没有这个属性呢？

那就会去原型对象的原型对象上查找，如果还没有，就去原型对象的原型对象的原型对象上查找···

const obj1 = {}

const obj2 = {}

const obj3 = {
  name: 'obj3',
}

obj1.__proto__ = obj2
obj2.__proto__ = obj3

console.log(obj1.name) // obj3

其实这就是原型链的查找过程。

# Object 的原型对象

那这样的查找没有尽头吗？obj3 的原型对象是谁呢？

console.log(obj3.__proto__) // [Object: null prototype] {}

console.log(obj3.__proto__.__proto__) // null

可以看见，obj3 的原型对象是一个空对象，这个空对象的原型对象是 null。

其实 [Object: null prototype] {}，这个对象就是顶层原型了，它的原型是 null，所以查找的过程就到此为止了。

[Object: null prototype] {} 这个对象就是 Object 的原型对象，也就是说，Object 的原型对象的原型对象是 null。

console.log(Object.prototype.__proto__) // null

我们平时用的最多的字面量方式创建的对象，其实本质都是通过 Object 构造函数创建的，所以它们的原型对象都是 Object.prototype。

const obj1 = {}

const obj2 = new Object()

console.log(obj1.__proto__ === Object.prototype) // true
console.log(obj2.__proto__ === Object.prototype) // true

console.log(obj1.__proto__) // [Object: null prototype] {}
console.log(obj1.__proto__.__proto__) // null

Object 构造函数的原型对象有什么特殊吗？

  1. 它是顶层原型，它的原型是 null
  2. 它身上有很多实用的方法，比如 toString、valueOf、hasOwnProperty···

所以什么是原型链：

原型链是由一系列对象组成的，每个对象都与一个原型对象相关联，形成了原型链。当访问对象的属性时，如果对象本身没有该属性，JavaScript 引擎会顺着原型链向上查找，直到找到匹配的属性或到达原型链的末尾。

这种机制使得 JavaScript 中共享属性和方法变得容易，也使得 JavaScript 可以像面向对象语言一样使用。

# 继承

面向对象有三大特性：封装、继承、多态。

  • 封装：将数据和操作数据的方法绑定在一起，数据被保护在内部，外部只能通过暴露的方法来操作数据，这样可以防止外部随意修改数据，导致数据不可控。
  • 继承：子类继承父类的属性和方法，子类可以重写父类的方法，也可以添加新的属性和方法。
  • 多态：同一个方法可以根据不同的对象，表现出不同的行为。

我们只关注继承，继承的目的是为了代码的复用，减少重复的代码。
