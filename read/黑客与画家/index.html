# 书摘与想法 - 黑客与画家

《黑客与画家》，作者是 Paul Graham（保罗·格雷厄姆），是一本充满启发性的书籍，作者在其中提出了许多有趣而超前的观点。

编程语言的一个重要特点：一个操作所需的代码越多，就越难避免 bug，也越难发现它们。

我觉得这句话很有道理，所以我一直在思考，如何让代码更少、更简洁、更优雅。

可能因为想炫耀自己见多识广，某些黑客会告诉你所有高级语言基本相似。“所有编程语言我都用过。”某个看上去饱经风霜又酷的黑客往酒吧里一坐，“你用什么语言并不重要，重要的是你对问题是否有正确的理解。代码以外的东西才是关键。”这当然是一派胡言。各种语言简直是天差地别。

我经常听见身边的人说，编程语言都差不多，掌握一门，其他的都是大同小异之类的话。我觉得这是一种误解，编程语言的差异性是很大的，它们有不同的设计理念、不同的编程范式、不同的语法特性、不同的应用场景···，我的观点和作者一样。

为了写出优秀软件，你必须同时具备两种互相冲突的信念。一方面，你要像初生牛犊一样，对自己的能力信心万丈；另一方面，你又要像历经沧桑的老人一样，对自己的能力抱着怀疑态度。

还有一个关键点是，要找到两种信念的平衡点。 如果过于自信，你就会写出糟糕的代码，因为你认为自己写的代码都是对的，不会去思考它是否有更好的实现方式； 如果过于怀疑，你就会写不出代码，因为你认为自己的想法和代码都是错的。

应该使用哪一种语言？嗯，关于这个问题，现在有很多争论。部分原因是，如果你长期使用某种语言，你就会慢慢按照这种语言的思维模式进行思考。所以，后来当你遇到其他任何一种有重大差异的语言，即使那种语言本身并没有任何不对的地方，你也会觉得它极其难用。

这是一种很有意思的现象，我也有这样的感觉，特别是当我学习 Dart 和 Rust 时，我就会觉得它们很难用，并不是觉得语言本身有什么问题，而是它们与 JavaScript 的差异性太大了。

让自己感到好受一些。当你踩水的时候，你把水踩下去，你的身体就会被托起来。同样，在任何社会等级制度中，那些对自己没自信的人就会通过虐待他们眼中的下等人来突显自己的身份。

这好吗？
